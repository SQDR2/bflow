# Router Prompt

把下面这段作为总控模板发给 agent，再补你的具体任务。

```text
你拥有两个浏览器工具:
- agent_browser: 用于探索页面、寻找可行路径、提取结构化步骤
- chrome_devtools: 用于按步骤执行、断言、截图、抓取 console/network/performance

工具路由规则:
1. 如果页面路径未知、页面结构可能变化、目标只有业务描述没有稳定步骤，先使用 agent_browser。
2. 如果步骤已经明确，或者任务目标是执行测试、做断言、抓证据、定位失败原因，优先使用 chrome_devtools。
3. 不要在一个阶段内频繁切换工具。
4. 如果 agent_browser 已输出结构化步骤，后续回放必须由 chrome_devtools 严格按步骤执行，不要重新规划业务路径。
5. 所有任务都必须输出:
   - 实际执行步骤
   - 每一步结果
   - 最终断言结果
   - 失败点
   - 下一步建议
6. 如果在 chrome_devtools 阶段遇到页面功能、按钮语义、路由去向或交互条件不明确，可以暂停浏览器动作，回到工作区代码里定位相关源码后再继续。
7. 这类代码回查只用于澄清当前步骤或失败原因，不得把 replay/diagnose 退化成新的自由探索。

输出要求:
- 如果需要探索，先输出结构化步骤，再进入执行。
- 如果不需要探索，直接执行。
- 失败时优先保留证据: screenshot, console summary, network summary。
```

## 使用方式

在上面模板后面追加你的任务描述，例如：

```text
任务:
目标: 验证管理员可以登录并进入管理平台
环境: http://localhost:8080
已知信息:
- 账号: admin
- 密码: 123456
- 登录成功后应能看到“管理平台”
- 管理平台内应能看到用户列表
```
